# Project Context

## 1. 项目名称

Small Shop Review Response & Insight Agent

## 2. 项目定位

面向线下小型门店的 AI Agent 工作流系统。用户上传顾客评论 CSV 后，系统自动完成评论校验、主题分类、情绪识别、问题聚合、回复草稿生成、安全检查、人工审批及 Trace/Eval 展示。

本系统不是完整 SaaS 平台或自动回复机器人，而是一个本地可运行、可解释、可审批、可评测的 Agentic Workflow。

## 3. 目标用户

独立咖啡店、餐厅等小型门店经营者。

用户特征：
- 无专门数据分析人员
- 关注差评但缺乏系统复盘时间
- 需要快速了解当前批次评论中的严重问题
- 需要可修改和发布的差评回复草稿

## 4. 核心闭环

```
CSV 上传 → 输入校验与清洗 → 评论分类 → 情绪与严重程度分析
→ Top 3 问题聚合 → 回复草稿生成 → 安全检查 → 人工审批
→ Dashboard 展示 → Trace & Eval 展示
```

## 5. 功能边界

### 范围内

- CSV 上传与校验
- 评论主题分类
- 情绪分析
- Top 3 问题聚合
- 差评回复草稿
- Safety Check
- 人工审批
- Dashboard
- Trace Log
- Eval Summary
- 内置数据离线运行

### 范围外

- 平台爬虫
- 自动发布回复
- 多门店系统
- 账号权限系统
- 移动端
- 复杂 BI Dashboard
- 周报生成
- 长期记忆
- 多 Agent 自治编排

## 6. UI 风格

浅色 SaaS 工作台 + 咖啡店暖色点缀。

设计关键词：干净、温和、可信、轻量、可操作、有证据感。

主色：#2F6B4F
辅助色：#D99A32
背景色：#F7F6F2
卡片色：#FFFFFF

## 7. 核心工程原则

1. LLM 不直接控制最终发布
2. 所有 LLM 输出须结构化
3. 所有洞察须绑定 review_id
4. 所有回复须经 Safety Check
5. 所有审批动作写入 Trace
6. 系统须离线稳定运行完整流程

# PROJECT_CONTEXT.md

## 1. 项目名称

Small Shop Review Response & Insight Agent  
小店差评处理与问题洞察 Agent

## 2. 项目定位

本项目是一个面向小型线下门店老板的 AI Agent MVP。老板上传顾客评论 CSV 后，系统自动完成评论校验、主题分类、情绪识别、三大问题聚合、差评回复草稿生成、安全检查、人工审批和 Trace/Eval 展示。

本项目不是完整 SaaS，也不是自动回复机器人。第一版目标是做一个本地可演示、可解释、可审批、可评测的 Agentic Workflow。

## 3. 目标用户

第一版默认用户为独立咖啡店老板或店长。

用户特点：

- 没有专门数据分析人员；
- 会关注差评，但没有时间系统复盘；
- 希望快速知道本批评论中最严重的问题；
- 希望获得可直接修改和发布的差评回复草稿。

## 4. MVP 核心闭环

上传 CSV
→ 输入校验与清洗
→ 评论分类
→ 情绪与严重程度分析
→ 聚合三大问题
→ 生成差评回复草稿
→ 安全检查
→ 人工审批
→ Dashboard 展示
→ Trace & Eval 展示

## 5. 第一版功能边界

### 必做

- CSV 上传
- 输入校验
- 评论主题分类
- 情绪分析
- 三大问题聚合
- 差评回复草稿
- Safety Check
- 人工审批
- 简单 Dashboard
- Trace Log
- Eval Summary
- Demo Mode

### 不做

- 平台爬虫
- 自动发布回复
- 多门店系统
- 完整账号权限
- 移动端 App
- 复杂 BI Dashboard
- 完整周报
- 长期记忆
- 复杂多 Agent 自治编排

关键词：

- 干净
- 温和
- 可信
- 轻量
- 可操作
- 有证据感

## 7. 核心工程原则

1. 不让 LLM 直接控制最终发布。
2. 所有 LLM 输出必须结构化。
3. 所有洞察必须绑定 review_id。
4. 所有回复必须经过 Safety Check。
5. 所有审批动作必须写入 Trace。
6. Demo Mode 必须可稳定跑完整流程。
